# 一问一视频自动生产标准块

这个标准块用于“用户只提出一个问题，Codex 自动生成可发布口播视频”的场景。目标不是给建议，而是自动完成：选题判断、文案、声音、逐句分镜、画面、动效、封面、导出和验收。

典型输入：

- “什么是劳动补偿金？”
- “什么是劳动赔偿金？”
- “为什么进口商许可证账号一定要在自己人手里？”
- “装修合同里最容易踩的坑是什么？”

## 目标结果

用户只给一个问题时，Codex 必须产出：

1. 60-120 秒中文口播文案。
2. 用 VoxCPM 或用户指定声音生成的口播音频。
3. 基于真实音频转写出来的逐句字幕时间轴。
4. 每句话对应的分镜总控表。
5. 稳定统一的竖屏机制动画视频。
6. 生图模型生成的封面。
7. 可发布 MP4，封面可以独立导出，也可以放在第一帧。

## 总流程

### 1. 理解问题和行业

先判断用户的问题属于哪一类：

- 法律解释：劳动补偿金、劳动赔偿金、离婚财产、合同违约。
- 合规证照：FDA、进口商许可证、公司注册、税务。
- 商业经验：换服务商、账号控制权、客户踩坑。
- 科普知识：一个概念、一个误区、一个机制。

不要等用户给行业资料。如果缺资料，Codex 先用常识搭建结构；如果涉及法律、政策、价格、时间敏感内容，必须联网查官方或可靠来源，再写文案。

### 2. 自动写口播文案

文案结构：

1. 开头 3 秒直接抛问题或冲突。
2. 用普通人能懂的话定义概念。
3. 说清楚适用条件。
4. 给一个真实感强的例子。
5. 拆出关键判断标准。
6. 给一句风险提醒。
7. 最后给可执行建议。

文案不要写成百科解释。要像真人在给老板、客户、普通人讲清楚一件事。

法律类问题的默认结构：

- 先说一句结论。
- 再说“它解决的是什么问题”。
- 再说“什么时候有，什么时候没有”。
- 再用一个例子区分相近概念。
- 最后提醒具体案件要看材料，不要做绝对承诺。

### 3. 生成音频

优先使用本地 VoxCPM。

如果用户有自己的声音样本：

- 先找用户指定或本机已有的 reference audio。
- 用 `scripts/check_voxcpm.py` 检查环境。
- 用 `scripts/voxcpm_synthesize.py` 生成音频。
- 如果有参考音频文字，使用 prompt audio + prompt text；没有文字时，先转写参考音频再使用。

如果没有用户声音：

- 使用 VoxCPM voice design 生成适合行业的声音。
- 法律/合规类默认：沉稳、清楚、可信、不夸张。
- 商业案例类默认：口语化、节奏稍快、带提醒感。

生成音频后，必须再转写一次。后续字幕、分镜、动效都以生成后的真实音频为准。

### 4. 逐句分镜

不要按 15 秒拆。按真实音频每句话拆。

每句话必须有：

- 时间范围。
- 原话字幕。
- 关键词。
- 观众此刻要看懂什么。
- 主视觉对象。
- 状态变化。
- 微动效。
- 验收帧。

每句话都要推动画面变化。没有变化的视频就是 PPT。

### 5. 选择主视觉模型

不同问题用不同视觉模型：

- 概念解释：定义卡 + 判断条件 + 对比表。
- 法律区别：左右对比 + 案例时间线 + 金额/责任流。
- 业务风险：主业务对象 + 权限流 + 阻断状态。
- 科普机制：对象变化 + 因果箭头 + 结果状态。

全片只保留一个主视觉系统，不要每句话换一种设计语言。

### 6. 做画面和动效

画面默认用代码图形做，不靠堆文字。

优先复用 `assets/remotion-v5-template/` 里的 V5 模板资产：

- `src/FdaLicenseV5Full.example.tsx`：完整 90 秒业务案例视频的画面结构、字幕层、安全区、视觉延迟、关键词动效和状态机示例。
- `src/FdaLicenseCover.example.tsx`：封面/第一帧结构示例。
- `package.example.json`：Remotion 依赖示例。

复用时不要照搬 FDA 文案。只保留布局、动效函数、字幕安全区、视觉延迟、状态机组织方式，把业务对象和分镜数据替换成当前行业内容。

必须做到：

- 字幕贴真实音频。
- 画面动效比声音慢 `0.3s-0.6s`。
- 关键词出现时有微动效。
- 底部 20%-25% 不放重要信息。
- 后半段不能退化成静态卡片。

### 7. 生成封面

封面用生图模型，不用 HTML 卡片。

封面要求：

- 一个清楚的人物/场景/冲突隐喻。
- 一个大标题。
- 一个小标签。
- 底部留白，避开平台遮挡。
- 不生成假官方证件、假法院文书、假印章、假平台页面。

法律类封面示例：

- “补偿金和赔偿金，差一个字，结果完全不同”
- 画面：一份劳动合同被分成左右两边，一边是正常离职，一边是违法解除。

### 8. 导出和验收

导出前必须检查：

- 视频时长和音频时长是否一致。
- 字幕是不是口播原话。
- 第一帧是不是封面。
- 每个关键词是否有画面响应。
- 画面是否左右居中。
- 字幕是否离主视觉太远。
- 底部抖音区域是否干净。
- 最后 30 秒是否仍然有逐句动效。

## 不允许

- 不允许只输出文案让用户自己做。
- 不允许用总结字幕代替真实字幕。
- 不允许画面只是一页页 PPT。
- 不允许生成假截图、假证书、假法院文书、假官方页面。
- 不允许因为换行业就换掉整个视觉质量标准。
- 不允许没有音频就拍脑袋定时间轴。

## 一句话标准

用户给一个问题，Codex 要把它变成“真实声音驱动的、逐句有画面反应的、可发布的竖屏口播视频”。
